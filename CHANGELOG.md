# Changelog

记录每次发布的变更内容。

## [1.2.0] - 2026-07-16

### 架构重构

- 主 Agent 正式命名为“创造者（Creovator）”，增加目的拆解、事实/推断/未知分层与渐进式高价值追问规则。
- 将单线交付重构为“0→1 产品、产品变更、维护执行”三条路线；维护任务使用轻量任务契约，不再强制生成 Product Spec 和 DEV-PLAN。
- 增加路线升级规则：正确行为不清或涉及公开契约、数据、权限与用户流程变化时，由维护执行升级到产品变更。
- 统一 R0/R1/R2 风险分级；产品 Phase 与 R2 变更要求 fresh reviewer，低风险维护采用相称验证。
- 将“AI 优先”修正为“AI 适用性评估”，避免为普通自动化强塞 AI 功能。

### 自进化与设计

- 重构自进化闭环：明确信号采集事件、合并键、严重度与五种状态，普通信号保留观察并累计，严重事件可单次触发。
- Codex 与 Claude Code 发布包现在都会预建 evolution/signals.md，更新工作流时保留已有台账。
- 统一产品文档默认路径为 `docs/`，同时兼容旧项目根目录文件。
- 将“设计绝对覆盖需求”改为按职责裁决，避免视觉稿篡改功能范围。
- 移除审美词到固定皮肤的映射，改为结合任务、内容、用户、环境与可访问性生成可验证设计决策。

### 构建与发布

- `release-builder` 改为识别 Node、Python、Rust、Go、Java、.NET、移动端和桌面端生态，不再假设 npm。
- 发布包新增 `.creator-manifest.json`，Skill 数量由源码目录动态生成；gateway 不再覆盖包内语义版本。
- 构建器严格校验每个 Skill 的命令映射和 `openai.yaml` 必填字段，禁止静默跳过不完整入口。
- 发布验证升级为失败即阻断：检查双平台结构、版本、manifest、Claude 转换残留、命令数量与 gateway 包结构；GitHub Release 在创建前执行同一验证。
- 修复 `reviewer` frontmatter 中未转义冒号导致严格 YAML 解析失败的问题。

## [1.1.0] - 2026-06-23

### 优化

- **技能描述重构**：全面重构优化了 11 个原子技能的 `SKILL.md` frontmatter 描述与 `openai.yaml` 触发配置。
- **降本增效防误触**：统一使用“定位+动作+产物边界”的描述模板，拉开开发（dev-builder）、修复（bug-fixer）、审查（reviewer）等重度执行技能的特征词区分度，将前置依赖门禁显性化，极大提升大模型工作流路由（Routing）成功率。

## [1.0.2] - 2026-06-23

### 优化

- **纯净版发布**：彻底移除所有对外部产品的引用痕迹与开发残留，优化 Git 提交历史为全中文。
- 更新构建脚本触发逻辑。

## [1.0.1] - 2026-06-23

### 重大升级：全线技能硬核化 (Hardcore Mode)

- **主线核心重制**：
  - `product-spec-builder`：引入 0-1 与迭代双模式，强制 6 大维度信息搜集门槛。
  - `design-brief-builder`：实行“二选一 (This-or-That) 引导法”，废除开放式主观提问。
  - `dev-planner`：实行无占位符原则与洋葱剥皮拆解法，强制精确到文件级交付。
  - `dev-builder`：引入 **“验证即证据 (Verification is Evidence)”** 与 **“强制独立交叉审查 (Independent Review)”**，代码必须当场通过编译/运行，且必须召唤 `reviewer` 子代理放行后才可宣称完成。

- **外围支撑重制**：
  - `bug-fixer`：引入“四阶段系统性调试法”，并增加【3 次失败强行阻断机制】防死循环。
  - `release-builder`：内置严苛的终端【隐私扫描流水线】，强制拦截包含开发者本地路径、密钥、DB 的打包行为。
  - `design-maker`：实行“组件先行”与“状态完备（加载、空态等）”强制校验。
  - `skill-builder`：实行“三层模块化法则”，系统后续自建 Skill 必须自带红线与门禁。
