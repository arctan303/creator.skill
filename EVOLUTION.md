# 自进化协议

## 目的

自进化系统负责把真实使用中的问题转成对 Skill 的补丁，而不是无限新增规则。

它要解决两件事：

- 补上反复出现的缺口
- 删除或改写长期无用、造成摩擦的规则

## 信号来源

只有真实使用信号才进入自进化：

- 用户纠正
- 用户明确不满意
- Agent 多次误解同类意图
- Skill 漏掉关键验证
- 审查发现本该由前置 Skill 防住的问题
- 某条规则经常造成不必要阻塞
- 某条规则长期没有发挥作用

不要因为理论上“可能更好”就修改 Skill。

## 记录位置

原始信号记录在：

```text
.codex/evolution/signals.md
```

信号只记录问题和上下文，不在当场直接改规则。

## 处理时机

自进化默认在下一次新开话题时处理。

处理时由 `self-evolver`：

1. 读取信号
2. 判断问题属于哪个 Skill
3. 找到对应 Skill 的现有规则
4. 判断是新增、修改、合并还是删除
5. 生成补丁
6. 等用户确认后应用

## 补丁原则

补丁是编辑，不是堆叠。

每次补丁都要回答：

- 当前哪条规则没覆盖这个问题
- 是补新规则，还是改旧规则
- 有没有旧规则应该删掉或合并
- 修改后是否更短、更准、更容易执行

## 禁止事项

- 不在执行关键任务中途随手改 Skill
- 不把单次偶发问题升级成永久规则
- 不只新增不删除
- 不把用户偏好伪装成通用规则
- 不在未经用户确认时应用规则变更
