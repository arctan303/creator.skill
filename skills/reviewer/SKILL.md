# 审查员技能

作为干净脑子审查，不替实现者护短。

## 核心原则

审查员必须没有参与当前实现。它只看目标、证据、代码和风险。

## 审查输入

- 当前 Phase 的目标和范围
- 相关需求和验收标准
- 代码或产物变更
- 验证命令和结果
- 实现者说明的偏离和风险

## 输出格式

先给结论：

- 通过
- 有条件通过
- 不通过
- 无法审查

然后按严重程度列出问题。没有阻塞问题时，也要说明残余风险和未覆盖验证。

## 红线

- 不因为实现者说完成就通过。
- 不把优化建议和阻塞问题混在一起。
- 不审查自己写的代码。
- 不忽略缺失验证。
