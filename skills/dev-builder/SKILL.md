# 开发技能

根据开发步骤文档完成一个 Phase，并证明它真的完成。

## 核心原则

任务不是因为写了代码就完成。只有验证证据和干净子代理审查都支持验收标准时，才算完成。

## 工作方式

1. 读取 `DEV-PLAN.md`、相关需求和设计文档。
2. 确认当前目标 Phase 和验收标准。
3. 只实现当前范围内的内容。
4. 运行必要验证。
5. 调用 fresh 子代理审查。这个子代理不能是写代码的同一个脑子。
6. 审查不通过就修，直到通过或记录明确阻塞。
7. 输出变更摘要、验证证据、审查结果和残余风险。

## 交付证据必须包含

- 修改过的文件或产物
- 验证命令和结果
- 覆盖了哪些验收标准
- 子代理审查结果
- 相对计划的偏离和原因

## 合格标准

当前 Phase 合格的标准是：

- 实现符合计划范围
- 必要检查通过，或阻塞被明确记录
- fresh 子代理审查通过
- 用户可以根据证据判断完成情况

## 红线

- 不静默修改已确认需求、设计或计划。
- 不跳过验证。
- 不自审冒充独立审查。
- 不扩大范围来伪装完成度。
- 不隐藏失败检查或跳过的测试。
