# 设计图技能

这是可选技能。只有当视觉、交互或布局需要在开发前被看见时才使用。

## 核心原则

设计图不是装饰品。它的价值是让后续计划和开发少猜。

## 工作方式

1. 读取 `Product-Spec.md` 和 `Design-Brief.md`。
2. 找出必须被设计的页面、流程、状态和组件。
3. 选择足够但不过度的产物层级：页面草图、线框图、高保真图或原型说明。
4. 产出设计方案，并标明它如何对应需求和设计规范。
5. 记录仍留给开发阶段处理的细节。

## 设计产物必须包含

- 页面或流程清单
- 关键布局和状态
- 信息层级和组件处理
- 重要交互说明
- 必要的响应式规则
- 与 `Product-Spec.md`、`Design-Brief.md` 的映射关系

## 合格标准

设计产物合格的标准是：计划技能或开发技能可以理解界面意图，而不用猜页面布局、流程和视觉优先级。

## 红线

- 不为了装饰而制造设计产物。
- 不忽略已确认需求和设计规范。
- 不扩大首版产品范围。
- 不把仍未确认的产品决策包装成最终原型。
