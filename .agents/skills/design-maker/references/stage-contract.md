# 设计图技能阶段契约

## 输入

- `Product-Spec.md`
- `Design-Brief.md`
- 用户提供的参考图、截图、品牌资产或原型资料

## 输出

可能输出：

- 设计方案
- 线框图
- 高保真原型说明
- 面向实现的 UI 注释

使用能降低开发不确定性的最小设计产物集合。

## 触发条件

只有满足至少一个条件时才启用：

- UI、布局、交互或品牌表达会显著影响开发计划。
- 用户需要在开发前检查关键页面或状态。
- `Design-Brief.md` 仍不足以让开发者判断界面结构。
- 目标项目需要对外部设计工具或视觉产物进行交接。

可以跳过的情况：

- 产品没有明显界面。
- UI 足够常规，开发计划可以直接拆分。
- 设计产物只会增加装饰，不减少风险。
- 产品流程或设计规范还不稳定，应回到前序阶段。

## 最小设计产物

设计产物至少说明：

- 页面或流程清单
- 关键布局
- 空状态、加载状态、错误状态、成功状态
- 关键交互
- 响应式规则
- 与 `Product-Spec.md` 和 `Design-Brief.md` 的映射关系
- 留给开发阶段处理的细节

## 合格标准

设计产物必须：

- 覆盖首版中需要视觉决策的关键流程
- 包含重要空状态、加载状态、错误状态和成功状态
- 体现 `Design-Brief.md`
- 能被计划技能用于拆分开发任务
- 不擅自扩展产品范围

## 必须回到前序阶段的情况

- 必要页面还没定义
- 风格方向仍不具体
- 产品流程不稳定
- 当前设计产物只是视觉装饰，不能减少决策风险

## 必须打回的情况

- 设计需要新增产品范围。
- 关键流程没有在 `Product-Spec.md` 中确认。
- 风格方向没有在 `Design-Brief.md` 中具体化。
- 用户要求高保真产物，但没有足够品牌、内容或布局依据。
