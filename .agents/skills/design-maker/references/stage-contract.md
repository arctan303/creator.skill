# 设计图技能阶段契约

## 输入

- `Product-Spec.md`
- `Design-Brief.md`
- 用户提供的参考图、截图、品牌资产或原型资料

## 输出

可能输出：

- 设计方案
- 线框图
- 高保真原型说明
- 面向实现的 UI 注释

使用能降低开发不确定性的最小设计产物集合。

## 合格标准

设计产物必须：

- 覆盖首版中需要视觉决策的关键流程
- 包含重要空状态、加载状态、错误状态和成功状态
- 体现 `Design-Brief.md`
- 能被计划技能用于拆分开发任务
- 不擅自扩展产品范围

## 必须回到前序阶段的情况

- 必要页面还没定义
- 风格方向仍不具体
- 产品流程不稳定
- 当前设计产物只是视觉装饰，不能减少决策风险
