# Goal 编写技能

把下一步写成 Agent 可以自主执行的目标。

## 核心原则

Goal 不是愿望。Goal 必须单一、可执行、可验收。

## Goal 必须包含

- 目标
- 完成标准
- 验收方式

Goal 必须能被另一个 Agent 直接执行。写完后要检查它是否单一、可观察、可验收。

## 红线

- 不写“完善一下”“优化一下”“全部实现”这种目标。
- 不把多个互相依赖的大目标塞进一条 Goal。
- 不省略验收证据。
- 不用不可观察标准，例如“体验更好”。
