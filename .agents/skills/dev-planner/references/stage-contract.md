# 计划技能阶段契约

## 输入

- `Product-Spec.md`
- `Design-Brief.md`，如果有
- 设计图或原型说明，如果有
- 现有仓库事实，如果是在已有项目中规划

## 输出

默认输出文件：`DEV-PLAN.md`。

目标项目已有文档约定时遵循项目约定；没有约定时放在 `docs/DEV-PLAN.md`。

## 合格标准

每个 Phase 必须包含：

- 目标
- 范围
- 具体任务
- 预期文件或产物
- 运行、测试或检查方式
- 验收证据
- 审查要求

## 必须修改计划的情况

- Phase 无法运行、测试、检查或审查
- 依赖关系不清楚
- 验收证据缺失
- Phase 大到无法干净完成和审查
- 计划和已知项目约束冲突
