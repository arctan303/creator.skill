# 自进化阶段契约

## 输入

- `EVOLUTION.md`
- `.codex/evolution/signals.md`
- 相关 Skill 的当前内容

## 输出

- 补丁建议
- 被修改规则的理由
- 删除或合并规则的理由
- 需要用户确认的问题

## 信号判定

可处理信号必须来自真实使用，例如：

- 用户纠正
- 多次误解
- 多次漏验证
- 审查发现本该由前置 Skill 防住的问题
- 某条规则造成明显摩擦
- 某条规则长期无用

不能处理的信号：

- 只有理论偏好，没有真实使用证据
- 单次偶发且影响很小
- 用户临时偏好被误当成通用规则

## 补丁策略

处理顺序：

1. 先找现有规则是否可修改。
2. 能合并就合并。
3. 能删除无用规则就删除。
4. 只有现有规则无法覆盖时才新增。

## 低风险补丁

以下通常可以按项目规则直接应用：

- 澄清触发条件。
- 补充验收证据要求。
- 合并重复规则。
- 删除明显过时或无用规则。
- 修正与 AGENTS.md 明显一致的措辞。

## 必须用户确认的变化

- 改变主流程顺序。
- 降低交付、审查、隐私或发布标准。
- 改变权限、安全或真实发布边界。
- 大范围跨多个 Skill 重写。
- 信号证据不足但影响较大。

## 信号处理结果

处理后必须把信号状态改为：

- `已转补丁`
- `已忽略`

并保留简短处理说明。
