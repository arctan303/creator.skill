# 自进化技能

把真实问题变成 Skill 补丁。目标是越用越准，不是越写越胖。

## 核心原则

自进化是编辑现有规则，不是无限新增规则。

## 工作方式

1. 读取 `EVOLUTION.md` 和 `.codex/evolution/signals.md`。
2. 按信号判断涉及哪个 Skill。
3. 阅读对应 Skill 的现有规则。
4. 判断应该新增、修改、合并还是删除。
5. 生成补丁建议。
6. 等用户确认后再应用。

## 红线

- 不在执行关键任务中途改规则。
- 不把单次偶发问题升级成永久规则。
- 不只新增不删除。
- 不未经用户确认直接修改 Skill。
