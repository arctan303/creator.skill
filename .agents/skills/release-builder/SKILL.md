# 发布技能 (release-builder)

把开发完成的代码转成可验证、可追溯、可回滚的交付物。未经产物验证与敏感信息审计，不得声称具备发布条件。

## [依赖检测]

启动前检查：
1. 必须有代码存在（如果是刚初始化的空项目，拒绝发布）。
2. 识别构建生态、版本来源、目标产物和目标平台；找不到可复现构建入口时标记阻塞。
3. 区分“生成本地发布包”和“真实推送/部署”。真实发布属于外部状态变更，必须取得用户明确授权。
4. 如果用户选择自动部署，检测相应 CLI 或平台认证状态；未认证则停止并指导用户操作，禁止索取真实密钥值。

## [生态识别与命令选择]

先读取仓库说明、CI 配置和项目脚本，再按实际标志文件选择工具链：

- Node/Web：`package.json` 与锁文件，使用仓库声明的包管理器和 build/test/audit 脚本。
- Python：`pyproject.toml`、`setup.cfg` 或 `setup.py`，构建 wheel/sdist，并在干净虚拟环境或 `pipx` 中安装冒烟。
- Rust：`Cargo.toml`，使用 `cargo test/build`，CLI 可用 `cargo install --path` 或产物二进制冒烟。
- Go：`go.mod`，使用 `go test/build`，对生成二进制执行冒烟。
- Java/Kotlin：`pom.xml` 或 Gradle 文件，使用 Maven/Gradle Wrapper 的 test/package 任务。
- .NET：`.sln`、`.csproj`，使用 `dotnet test/publish`。
- 移动端/桌面端：使用项目已有的 Gradle、Xcode、Electron、Tauri 或原生打包配置，并明确签名、商店后台和人工安装限制。

多生态仓库按发布目标分别记录命令和产物。仓库已有 Wrapper、锁文件或 CI 命令时优先复用；不得凭习惯替换包管理器或构建系统。

## [第一性原则]

1. **隐私与秘密保护**：发布产物不得包含开发者本地路径（如 `/Users/xxx`、`C:\Users\xxx`）、本地测试数据库、环境变量文件（如 `.env.local`）、用户数据或硬编码密钥。
2. **开发模式跑通 ≠ 产物可用**：必须运行仓库真实的生产构建或打包命令，并从最终产物执行安装、启动或接口冒烟。
3. **平台化验证**：
   - 桌面端/移动端：能本地安装时执行安装冒烟；依赖签名、设备或商店后台时明确人工验证和残余风险。
   - Web 端：获得部署授权且已有目标环境时验证线上产物和环境配置；仅做本地发布准备时记录尚未执行的线上验证。
   - CLI：按生态使用临时全局安装、干净环境安装或最终二进制运行，验证 `--help`、版本和核心命令。
4. **证据优先**：先读取完整错误、仓库配置和本地工具版本。问题依赖当前平台规则、签名要求或工具版本时，再查官方文档、发行说明或上游 Issue，并记录适用版本。

## [产物敏感信息审计]

生产构建完成后，在推送到远程或打包压缩前，必须对实际产物目录执行以下扫描：

1. **查开发者路径**：搜索是否有 `/Users/` 等个人绝对路径被打包了进去。
2. **查数据库**：搜索产物里有没有 `.db` 或 `.sqlite` 结尾的文件（防止用户数据或开发测试数据泄露）。
3. **查环境变量**：搜索产物里有没有包含真实的 `.env` 文件。
4. **查密钥泄漏**：全局搜索特征字符串，如 `sk-ant-`、`sk-proj-`、`OPENAI_API_KEY` 等硬编码信息。

**阻断机制**：扫描结果是候选证据，必须检查命中内容。真实秘密、个人绝对路径、环境文件、用户数据或测试数据库一经确认立即阻断；仅命中变量名、文档示例或脱敏占位符时记录判定理由，不制造误报。修复后重新构建并重扫。

## [发布检查清单]

在隐私审计通过后，执行以下检查：

- [ ] **版本号更新**：确认生态对应的版本来源（如 package、pyproject、Cargo、程序集或移动端配置）已更新并彼此一致。
- [ ] **依赖风险检查**：运行仓库或生态可用的 audit/漏洞检查；没有可靠工具时明确写出未检查范围，不伪造“无漏洞”。
- [ ] **工作区范围**：运行 `git status`，确认只有本次已知变更；真实发布前应提交或明确记录所有计划内未提交内容，任何意外文件都要阻断排查。
- [ ] **产物体积预估**：如果打包出了异常庞大的体积（如纯前端包出现几百 MB），必须暂停，打印出最大的文件让用户排查是否打入了不该打的内容（如 `node_modules`）。
- [ ] **安装与冒烟**：从最终产物而非源码目录验证安装、启动、版本输出或核心流程。
- [ ] **回滚路径**：说明如何恢复上一版本、撤销部署或停止分发。

## [输出规范]

完成部署/打包后，向用户输出一份专业的发布报告：
- **版本号**：本次发布的版本。
- **构建输出与大小**：明确给出构建目录和产物体积。
- **隐私审计结论**：说明扫描范围、候选命中和人工判定；只能声称在已检查范围内未确认泄漏。
- **发布产物位置/访问链接**：供用户下载安装包，或直接访问线上服务。
- **未覆盖范围与授权状态**：明确人工步骤、残余风险，以及是否获得真实发布授权。
