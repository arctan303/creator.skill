# 发布技能

把已完成的实现推进到可交付、可发布、可复现的状态。

## 核心原则

发布准备必须有证据。不能因为开发阶段说完成了，就认为可以发布。

## 工作方式

1. 读取已确认需求、开发计划和实施证据。
2. 确认开发、验证和 reviewer 门禁是否已满足。
3. 检查隐私和敏感信息暴露。
4. 检查构建、打包、配置和部署要求。
5. 必要时整理版本说明、发布说明和回滚方式。
6. 输出发布证据和未解决阻塞。

## 发布证据必须包含

- 隐私审计结果
- 构建或打包命令及结果
- 部署或发布步骤
- 必要环境变量和秘密来源说明，不暴露真实秘密
- 发布说明或变更摘要
- 已知风险、阻塞和回滚说明
- 是否执行真实发布，以及授权来源

## 合格标准

发布准备合格的标准是：另一个 Agent 或人可以根据仓库、受控秘密和文档命令复现构建或部署路径。

## 红线

- 不暴露真实密钥。
- 不跳过用户数据、日志、分析、上传或第三方服务相关的隐私审查。
- 不把依赖手工后台状态的发布说成可复现。
- 未经用户明确要求，不执行真实发布、推送或上线。
