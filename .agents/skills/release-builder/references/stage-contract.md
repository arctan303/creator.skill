# 发布技能阶段契约

## 输入

- 当前路线和对应交付契约：Product Spec + DEV-PLAN，或维护任务契约
- 实施证据、风险级别和审查结果
- 仓库构建、打包和部署配置
- 项目生态、版本来源、锁文件和 CI/Wrapper 命令
- 目标发布环境说明

## 输出

- 发布准备报告
- 隐私审计结果
- 构建、打包或部署证据
- 阻塞项和需要用户处理的动作
- 发布说明或变更摘要
- 回滚或恢复路径
- 是否执行真实发布的授权记录

## 发布证据模板

```markdown
# release-evidence.md

## 发布准备状态

状态：可发布 / 阻塞 / 未执行真实发布

## 审计范围

## 隐私和敏感信息审计

## 构建和打包检查

## 部署或发布路径

## 环境变量和秘密来源

## 发布说明

## 发布阻塞项

## 回滚或恢复路径

## 结论
```

## 隐私审计检查项

必须检查并说明：

- 真实密钥、token、password、private key 是否出现在仓库或输出中。
- 用户数据、日志、上传文件、分析事件、第三方服务调用是否被涉及。
- 截图、二进制、示例数据或归档资料是否可能含敏感信息。
- 发布证据是否避免打印真实秘密。
- 无法检查的范围和残余风险。

建议文本扫描包含：

```powershell
rg -n "(?i)(api[_-]?key|secret|token|password|passwd|private[_-]?key|BEGIN RSA|BEGIN OPENSSH|AKIA|sk-[A-Za-z0-9]|ghp_[A-Za-z0-9]|xox[baprs]-|client_secret)" .
```

## 构建、打包和部署证据

必须记录：

- 构建或打包命令。
- 选择该命令的生态与仓库依据。
- 命令结果，包含成功、失败或无可用命令。
- 产物路径，如果有。
- 部署步骤或发布步骤。
- 依赖的环境变量、平台配置和秘密来源。
- 哪些步骤可复现，哪些依赖人工后台状态。

没有构建配置时，必须写成阻塞或“不适用并说明原因”，不能声明可复现构建。

## 秘密来源说明

允许写：

- secret 名称
- 注入位置，例如 CI Secret、平台 Secret Store、本机环境变量
- 哪个步骤需要该 secret

禁止写：

- secret 的真实值
- token 的完整值
- 私钥内容
- 可用于登录或调用服务的凭据

## 合格标准

发布准备必须验证：

- 没有明显敏感信息暴露
- 配置和秘密来源被安全说明
- 构建或打包路径可运行，或阻塞明确
- 最终产物经过与生态匹配的安装、启动或接口冒烟
- 部署步骤可复现
- 必要时有回滚或恢复路径
- 当前路线要求的验证和 reviewer 门禁已通过，或未通过项被列为发布阻塞
- 真实发布、推送或上线有用户明确授权

## 禁止发布的情况

- 真实秘密被提交或打印
- 用户数据处理不清楚
- 构建或打包失败
- 部署依赖未记录的手工状态
- 当前交付契约要求的测试或审查门禁缺失
- 用户没有明确授权真实发布、推送或上线

## 阻塞项格式

```markdown
- BLOCK-01：<阻塞事实>。影响：<为什么阻止发布>。需要：<用户或后续阶段动作>。
```

## 必须打回的情况

- 开发/维护证据缺失，或要求的 reviewer 未通过。
- 无法判断是否有敏感信息暴露。
- 构建、打包或部署步骤无法复现且没有阻塞说明。
- 需要真实发布但没有用户授权。
- 发布证据中包含真实秘密。
