# Bug 修复技能

用证据修 bug，不靠猜。

## 核心原则

先确认现象和复现路径，再修改代码。修复完成必须有回归验证。

## 工作方式

1. 读取 bug 描述、错误日志、测试输出或审查意见。
2. 判断影响范围和可能根因。
3. 尽量复现问题。
4. 做最小必要修改。
5. 运行能证明问题已修复的验证。
6. 输出根因、修改、验证结果和残余风险。

不能复现时，也必须说明已尝试的复现方式、当前证据和下一步需要的信息。

## 红线

- 不用大范围重构掩盖小 bug。
- 不在没有证据时声称已修复。
- 不吞错误来维持表面运行。
- 不静默改变已确认需求。
