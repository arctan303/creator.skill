# Reviewer 阶段契约

## 输入

- 路线、风险级别及依据
- 目标、范围和非目标
- 产品文档与 Phase，或维护任务契约
- 变更文件和 diff
- 验证命令、结果、失败/跳过项和残余风险
- 审查指纹：契约版本、diff 标识和验证证据标识
- 前一轮结论与修复摘要（聚焦复审时）

## 收敛规则

- 相同审查指纹只保留一份 fresh reviewer 结论，不重复启动审查。
- 首轮应一次列出所有可发现问题，实施者批量修复。
- 修复产生新 diff 后最多启动一次聚焦复审，只核验原问题与受影响邻域。
- 同一阻塞问题连续两轮仍存在时停止继续复审，报告证据、影响和所需决策。
- 模型容量、网络、工具或宿主中断不算审查轮次；记录恢复检查点后停止新建 reviewer。

## 必查项

- 路线是否正确；维护是否掩盖产品变化
- 风险是否被低估
- 当前契约和验收标准是否被证据覆盖
- 是否有行为回归、范围扩大或设计偏离
- 是否缺少关键测试或验证
- 是否有安全、隐私、数据和发布风险
- 证据是否可复现

## 输出模板

```markdown
路线：0→1 产品 / 产品变更 / 维护执行（维持 / 升级）
风险级别：R0 / R1 / R2（维持 / 升级）
结论：通过 / 有条件通过 / 不通过 / 无法审查 / 中断
审查指纹：<contract + diff + evidence>

## 问题
- <严重级别>：<证据、影响、建议>

## 验收覆盖
- <REQ/AC/Phase/维护条件>：已覆盖 / 未覆盖 / 无法判断

## 残余风险
- <即使通过仍需知道的风险>
```

## 必须打回

- 新能力或外部行为变化被标为维护。
- R2 没有 fresh reviewer 独立性。
- 目标、范围、契约或 diff 不清。
- 实现者没有提供必要验证命令和结果。
- 验收标准没有映射证据。
- 发现阻塞问题但要求直接通过。

## Examples

### 同一 diff 去重

审查指纹未变化且已有结论时，返回现有结论和未处理问题，不创建第二个 reviewer。

### 新 diff 聚焦复审

实施者批量修复首轮问题并产生新 diff 后，只启动一次聚焦复审；若相同阻塞问题仍存在，停止循环并交还实施者或用户决策。

### 容量中断

出现 “Selected model is at capacity” 时输出“中断”，保存已完成检查、未完成范围和恢复入口；不得写成“无法审查”，也不得换一批 reviewer 反复尝试。
