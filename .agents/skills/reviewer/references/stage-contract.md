# Reviewer 阶段契约

## 输入

- 路线、风险级别及依据
- 目标、范围和非目标
- 产品文档与 Phase，或维护任务契约
- 变更文件和 diff
- 验证命令、结果、失败/跳过项和残余风险

## 必查项

- 路线是否正确；维护是否掩盖产品变化
- 风险是否被低估
- 当前契约和验收标准是否被证据覆盖
- 是否有行为回归、范围扩大或设计偏离
- 是否缺少关键测试或验证
- 是否有安全、隐私、数据和发布风险
- 证据是否可复现

## 输出模板

```markdown
路线：0→1 产品 / 产品变更 / 维护执行（维持 / 升级）
风险级别：R0 / R1 / R2（维持 / 升级）
结论：通过 / 有条件通过 / 不通过 / 无法审查

## 问题
- <严重级别>：<证据、影响、建议>

## 验收覆盖
- <REQ/AC/Phase/维护条件>：已覆盖 / 未覆盖 / 无法判断

## 残余风险
- <即使通过仍需知道的风险>
```

## 必须打回

- 新能力或外部行为变化被标为维护。
- R2 没有 fresh reviewer 独立性。
- 目标、范围、契约或 diff 不清。
- 实现者没有提供必要验证命令和结果。
- 验收标准没有映射证据。
- 发现阻塞问题但要求直接通过。
