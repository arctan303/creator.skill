# 开发技能阶段契约

## 输入

- `DEV-PLAN.md` 中的当前 Phase
- `Product-Spec.md`
- `Design-Brief.md` 或设计产物，如果相关
- 仓库代码和现有测试

## 输出

- 实现变更
- 验证证据
- fresh 子代理审查结果
- 进度记录或交接摘要

## 合格标准

开发结果必须：

- 满足映射到当前 Phase 的需求和验收标准
- 能通过说明的命令构建、测试、运行或检查
- 通过 fresh 子代理审查
- 明确记录任何设计偏离

## 子代理审查要求

子代理必须：

- 没有参与当前实现
- 获得足够上下文判断当前 Phase
- 优先检查 bug、漏需求、回归和缺失验证
- 输出明确通过或可执行的问题清单

## 必须继续闭环的情况

- 检查失败
- 验收标准未被证明
- 审查发现阻塞问题
- 变更超出计划范围
- 证据不可复现
