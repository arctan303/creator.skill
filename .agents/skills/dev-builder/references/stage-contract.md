# 开发技能阶段契约

## 可接受输入

### 产品路线

- Product Spec
- 完整或增量 DEV-PLAN
- Design Brief / 设计产物（如果相关）
- 当前 Phase、需求和验收标准

### 维护路线

- 维护目标和预期行为
- 当前证据
- 范围和非目标
- 完成证据与验证命令
- R0/R1/R2

R0/R1 短任务可以使用会话内契约；R2、跨会话、多阶段或多人交接任务使用 `docs/maintenance/<task-id>.md`。

## 输出

- 实现变更和文件清单
- 当前路线、契约映射、风险级别和依据
- 验证命令与结果
- R2 或产品 Phase 的 fresh reviewer 结果
- 偏离、阻塞和残余风险

## 合格标准

- 修改没有越出契约范围。
- 产品 Phase 映射需求、验收标准和计划。
- 维护任务证明预期外部行为未被意外改变。
- 验证可以复现并覆盖主要回归风险。
- R2 或产品 Phase 已通过 fresh reviewer；宿主不支持时明确记录限制，不伪造结论。
- 公开行为变化已升级到产品变更路线。

## 风险分级

- R0：纯文本、注释、文案、格式或不影响运行的结构变化。
- R1：局部、可回滚、边界明确，不涉及公开契约、跨模块、权限、数据、依赖或发布配置。
- R2：产品 Phase、跨模块或公开契约变化，以及认证、权限、支付、安全、隐私、迁移、依赖或部署变化。

不确定、验证不足或影响范围无法证明时按 R2。

## reviewer 上下文包

- 路线和目标契约
- 风险级别及依据
- 范围与非目标
- 需求/验收/变更 ID 或维护预期行为
- 变更文件和 diff
- 验证命令、结果、失败项和跳过项
- 已知偏离和残余风险

## 必须停止或升级

- 缺少可判断的目标契约。
- 维护任务需要重新定义正确行为。
- 修改超出范围或影响公开边界。
- 验证失败、不可复现或证据不足。
- R2 没有完成要求的独立审查。
