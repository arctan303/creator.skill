# 开发技能阶段契约

## 输入

- `DEV-PLAN.md` 中所有未完成且属于当前开发目标的 Phase 或任务
- `Product-Spec.md`
- `Design-Brief.md` 或设计产物，如果相关
- 仓库代码和现有测试

## 输出

- 实现变更
- 验证证据
- 由开发 Agent 自行启动的 fresh 子代理审查结果
- 每个 Phase 或任务的进度记录
- 项目开发完成结论，或明确阻塞

## 审查上下文包

开发 Agent 启动 reviewer 前，必须提供：

- 当前 Phase 或任务目标
- 对应需求编号和验收标准
- 计划范围和明确不包含的范围
- 变更文件或产物清单
- 验证命令和结果
- 已知偏离、失败检查、跳过验证和残余风险
- 希望 reviewer 判断的具体问题

## 合格标准

开发结果必须：

- 覆盖 `DEV-PLAN.md` 中所有计划内开发 Phase 或任务，除非阻塞被明确记录
- 每个完成的 Phase 或任务满足映射需求和验收标准
- 每个完成的 Phase 或任务能通过说明的命令构建、测试、运行或检查
- 每个完成的 Phase 或任务通过 fresh 子代理审查
- 全部计划内开发范围满足交付要求后，开发 Agent 才能表明项目开发结束
- 明确记录任何设计偏离

## 验证失败处理

- 检查失败时，不得继续请求 reviewer 通过。
- 能修复的失败必须先修复并重跑验证。
- 暂时无法修复时，必须记录阻塞原因、已尝试动作和下一步需要的输入。
- 跳过测试或验证必须说明原因和风险。

## 审查失败处理

- reviewer 发现阻塞问题时，必须按问题修复、重跑验证并复审。
- reviewer 有条件通过时，必须补齐条件后再记录完成。
- reviewer 无法审查时，必须补齐上下文或证据后重新审查。
- 不得把 review 建议静默忽略；非采纳项必须说明原因。

## 范围偏离处理

- 小的实现细节偏离可以记录在交付证据中。
- 影响需求、设计、公开行为、依赖或发布边界的偏离，必须回到用户或前序文档确认。
- 不得用扩大范围掩盖计划内任务未完成。

## 子代理审查要求

子代理必须：

- 没有参与当前实现
- 由开发 Agent 在完成实现和验证后主动启动
- 获得足够上下文判断当前 Phase
- 优先检查 bug、漏需求、回归和缺失验证
- 输出明确通过或可执行的问题清单

## 必须继续闭环的情况

- 检查失败
- 验收标准未被证明
- 审查发现阻塞问题
- 审查尚未启动或不是 fresh 子代理完成
- 仍有计划内开发 Phase 或任务未完成
- 变更超出计划范围
- 证据不可复现
